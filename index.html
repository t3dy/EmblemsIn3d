Systems & infrastructure
Tone.js 14.7.77

Alchemical Audio

Generative ambient soundscape per stage. NIGREDO: 3-stack sawtooth drone (42/43.2/40.8 Hz) + brown noise crackle. ALBEDO: 4 sine overtones with reverb decay=6 + tremolo. CITRINITAS: MetalSynth bell sequence + hum. RUBEDO: PolySynth triads + chorus. Unlocks on first click.

Three.js · BufferGeometry

ParticleStream

Quadratic bezier particle flows: B(t) = (1-t)²P0 + 2(1-t)tP1 + t²P2. Per-particle phase offset for staggered timing. Turbulence noise added (sin-based). AdditiveBlending, depthWrite:false. Used in every emblem scene and the HP fountain.

HP + AF databases

Data Pipeline

export_for_3d.py queries two SQLite databases. Dynamic column discovery via PRAGMA table_info. Outputs: emblems.json (51), hp_annotations.json, hp_symbols.json, hp_woodcuts.json, world_links.json (9 HP↔AF cross-refs with scholarly commentary).

BL marginalia

Annotation Panel

BL manuscript annotations surface after 6 seconds in any scene (or press A). findLinkedAnnotations() walks world_links.json to the HP folio, then checks hp_annotations.json. Falls back to 150-word editorial cross-reference commentary for folios not yet in the DB.

EffectComposer

Bloom Post-Processing

UnrealBloomPass strength varies by context: 0.35 (HP garden), 0.40 (gallery), 0.55–0.65 (Archives/emblems). ACESFilmic tonemapping + SRGBColorSpace output. Bloom calls made inside scene.build() so each scene self-configures the post-processing chain.

CSS transition

Fade Transitions

320ms CSS opacity 0→1→0 black overlay on every world switch. Keeps the Three.js render loop running during the transition — no flash or jump cut. All switchWorld() calls go through fadeSwitch(callback).

Tech stack
Rendering
Three.js r168 — ES module via jsDelivr CDN, loaded via importmap. EffectComposer + UnrealBloomPass. ACESFilmic tonemapping. SRGBColorSpace output.
Animation
GSAP 3.12.5 via esm.sh. gsap.timeline({ repeat: -1 }) for 12–16 second looping emblem timelines. Ease functions: sine.inOut, elastic.out, none.
Audio
Tone.js 14.7.77 via esm.sh — lazy-imported on first user gesture. OmniOscillator, NoiseSynth, MetalSynth, PolySynth, Reverb, Chorus, Tremolo, AmplitudeEnvelope, Loop, Transport.
Geometry
CapsuleGeometry (figures), SphereGeometry (heads/fire/nodes), TorusGeometry (rims/crowns/halos), CylinderGeometry (columns), OctahedronGeometry (stone), PlaneGeometry (wings/ground), BoxGeometry (hedges/cards), ConeGeometry (trees), LineSegments (rays), BufferGeometry (particles/curves).
Particles
THREE.Points + BufferGeometry. Quadratic bezier B(t) = (1-t)²P0 + 2(1-t)tP1 + t²P2. Per-particle phase offsets. AdditiveBlending. depthWrite: false.
Data
Two SQLite databases (AF emblems, HP manuscript). Python 3 export pipeline → 6 JSON files. No server-side rendering — all static. fetch('./data/emblems.json?v=8').
Deployment
GitHub Pages (static, branch: main, root). Python http.server 3456 for local dev. No bundler — native ES modules via importmap.
Games
Pure HTML/CSS/JS — zero dependencies. HTML5 drag & drop API, CSS perspective + rotateY 3D flip, IntersectionObserver entrance animations.
Creative decisions
Primitive-only geometry — no 3D model imports
Every figure and object is built from CapsuleGeometry, SphereGeometry, TorusGeometry, etc. No .glb or .fbx imports. This forces close reading of the emblem imagery — building the toad from a squashed sphere with emissive amber eyes is an interpretive act, not a texture lookup. It also keeps the project runnable with no asset pipeline.
Alchemical stage as the master design system
NIGREDO / ALBEDO / CITRINITAS / RUBEDO drive every visual decision: background colour, particle colour, bloom strength, audio synthesis approach, gallery card colour, Archives node colour, and even the annotation panel border. One four-value enum unifies 51 scenes, 4 games, and a graph.
Looping GSAP timelines (12–16s, repeat: -1)
Alchemical operations are cyclic — the Work has no terminal state. A 12-second loop encodes the philosophical claim that the prima materia is never destroyed, only transformed. The specific durations (12s for Nigredo scenes, 16s for Rubedo's completion emblem) give longer operations more time to develop.
Quadratic bezier particle paths
Every substance flow (milk, mercury, water, love-sparks) uses a bezier arc with a symbolic mid-point. The milk in Emblem V arcs upward before landing at the toad — it doesn't fall straight. The mid-point encodes the transformation: this substance is going somewhere, not just moving.
Bipartite graph for the cross-reference archive
The scholarship on these two texts is fundamentally about their network of visual and symbolic connections. A node-link diagram makes the argument structurally visible: HP folio nodes on the left, AF emblem nodes on the right, bezier edges connecting specific readings. Clicking a node navigates you there — the graph is executable bibliography.
Editorial commentary when the DB has no data
The British Library annotation data only covers folios 1–8, but all 9 world_links reference folios 14–164. Rather than show an empty annotation panel, we wrote ~150-word scholarly notes for each HP↔AF connection. These are grounded in the actual imagery and stored as first-class data in world_links.json — not hardcoded strings.
What we learned building this
Browser · iframe
window.innerWidth is 0 inside a preview iframe. Fixed with fallback chain: canvas.clientWidth || document.documentElement.clientWidth || window.innerWidth || 1280.
ES modules · TDZ
Calling resizeAll() at module-level before const composer and const bloom were declared put both in the Temporal Dead Zone. The init IIFE silently failed. Fix: define resizeAll after bloom, call it as the first line inside the async IIFE.
Browser · caching
Browsers aggressively cache ES modules as 304 Not Modified even when the file has changed. Version strings must be incremented on every file in the import chain: EmblemScene.js?v=3, Particles.js?v=3, main.js?v=8. A change in any leaf file requires bumping all importers.
Three.js · position
Object.assign(new THREE.Mesh(), { position: new THREE.Vector3() }) throws Cannot assign to read only property 'position'. Mesh.position is a getter returning an internal Vector3 — it's not a writable property. Always use mesh.position.set(x, y, z).
Windows · encoding
Python print statements with ✓ ⚠ ── characters throw UnicodeEncodeError on Windows cp1252 terminals. Replace all emoji/box-drawing chars with ASCII: OK, WARN, --.
Data · coverage
motto_latin is populated for only 1/51 emblems in the AF database. The memory game had to be redesigned from Latin↔English motto matching to Roman numeral↔English title matching. Check data coverage before designing games around a specific field.
Data · stage counts
CITRINITAS has 4 emblems, not 5. The stage-sorter assumed 5-per-stage (20 total). Fix: Math.min(5, pool.length) and dynamic denominator activeCards.length. Never hardcode stage counts — query them.
Data · SQL schema
The HP annotations table uses folio_number and annotation_text, not page and text. The export script now uses PRAGMA table_info(annotations) for dynamic column discovery rather than assuming column names.
GitHub Pages · paths
Absolute paths like /src/data/emblems.json break on GitHub Pages (they resolve to the domain root, not the repo subdirectory). All data fetches must be relative: ./data/emblems.json from src/, ../src/data/emblems.json from games/.

A digital humanities experiment: two Renaissance emblem books rendered as interactive 3D environments, cross-referenced in a live graph, and encoded as browser games. Built with Three.js, GSAP, and Tone.js — no bundler, no build step, no model assets.

Source code and data pipeline on GitHub. AF database from the Claudiens project. HP marginalia from the British Library copy (BL 1A.17757).
